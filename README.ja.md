# 🔥 神聖なる料理典籍

> *「すべての commit は一皿の料理。すべての merge は饗宴。すべての revert は失敗作。」*

---

## これは何か

料理の征服記録を**バージョン管理するリポジトリ**。プロダクションコードと同じ厳格さで運用される——食べ物にも CI/CD が必要だから。

料理ひとつにつきディレクトリひとつ。一日ごとにエントリーひとつ。フィードバックは commit される。失敗は記録される。何も記憶の entropy に消えることはない。

これはレシピブログではない。これは**インフラ**だ。

---

## リポジトリ構成

```
cooking-log.md              # インデックス。台帳。唯一の真実。
dishes/
  YYYY-MM-DD-<名前>/
    recipe.yaml             # 唯一のデータソース（多言語食材定義）
    zh.md                   # 中国語：手順・ポイント・フィードバック
    en.md                   # 英語テキスト
    ja.md                   # 日本語テキスト
```

---

## 規約

- 📐 **基準人数：2人前** — CI が ½、1、2、3、4、5人前の換算表を自動生成
- 🌿 食材の分量は正確に記載。「適量」は code smell
- 🔁 調理後にフィードバックを記録。レシピのパッチにつながることもある
- 🌐 すべてのレシピは中国語 / English / 日本語に対応
- ✅ すべての commit は署名済み。署名のない食べ物は食べ物ではない。

## 哲学
